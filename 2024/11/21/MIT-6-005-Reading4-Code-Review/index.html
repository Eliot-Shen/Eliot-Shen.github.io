MIT 6.005 Reading 4 Code Review

Code review really has two purposes:

  • Improving the code. Finding bugs, anticipating possible bugs, checking the clarity of the code, and checking for consistency with the project’s style standards.
  • Improving the programmer. Code review is an important way that programmers learn and teach each other, about new language features, changes in the design of the project or its coding standards, and new techniques. In open source projects, particularly, much conversation happens in the context of code reviews.

Don’t Repeat Yourself

Smelly example：

以上代码有很多重复的地方。如果你在两处或多处地方有相似或相同的代码，就会产生潜在风险，比如只修复了一处的bug而忘记其他地方。

小心使用CtrlC和CtrlV，你复制的代码越长，危险就越大。

个人认为最常出现重复之处在于函数重载Overload的时候，相似的函数体和功能，只有signature是不同的。

如果未来需要修改一处的功能或修复一处的bug，需要记得它还有个重载伙伴。

Comments Where Needed

注释的艺术。

1.方法和类上方的规范:Specification，以下为示例。

2.如果有从别处copy或改编过来的代码，请注明出处。

其中一个原因就是为了避免侵犯版权。

3.避免不必要的注释，比如直译代码意义，但要确保代码中有注释。

Fail Fast

尽早暴露出代码中的bug，最好让静态检查直接提示出错误所在，动态检查其次，最后才是运行时出错。

个人经验，使用logging模块记录程序运行中的输出。

Avoid Magic Number

可译为幻数, 其实就是硬编码, 编程时我们只能接受代码中出现0, 1, 最多2, 而不接受其他突兀的数字。

One Purpose For Each Variable

在dayOfYear函数中，对dayOfMonth做了加法用来计算一个含义完全不同的值，改变了其原本的含义。

不要重用参数和变量，变量不是编程中的稀缺资源。

注意，方法参数尤其要避免被修改。

在未来如果这个方法的某一部分想知道参数传进来的初始值，那么你就不应该在半路修改它。

Use Good Names

好的变量名可以自我解释，直接读懂其含义，无需额外注解。

Variable names like tmp , temp , and data are awful.

In Java:

  • methodsAreNamedWithCamelCaseLikeThis
  • variablesAreAlsoCamelCase
  • CONSTANTS_ARE_IN_ALL_CAPS_WITH_UNDERSCORES
  • ClassesAreCapitalized
  • packages.are.lowercase.and.separated.by.dots

方法名要使用动词，“ getDate or isUpperCase ”

用词简短但准确, 拒绝缩写, “ message is clearer than msg , and word is so much better than wd . ”

Smelly example：

方法名leap, 参数y, 变量tmp，糟透了！

Use Whitespace to Help the Reader

缩进的艺术，空格的艺术，Eclipse自带format功能！

另外不同的IDE环境，文本编辑器，对tab键的解析方式不同，tab可能等于2个空格，4个空格或8个空格。

为了代码的可移植性，请使用space或确保你的IDE将tab键等同于插入若干空格(修改Eclipse中的preference-Java-format-edit-(space-only))。

Don’t Use Global Variables

Smelly example：

public static但没有final，即为全局变量，public意味着它可以被任何地方访问，static意味着它跟任何类都没有关联，单纯只是一个变量。

我们应该将全局变量转为放入object中，或者放入参数和返回值。

参考网页-GlobalVariablesAreBad

Methods Should Return Results, not Print Them

Smelly example中只是打印了n，而非返回它。

一般来说，只有high-level的程序部分进行人机交互，low-level部分应该接受参数输入并返回结果。

唯一的例外就是调试部分，但调试部分不应该是你设计的功能的一部分。
